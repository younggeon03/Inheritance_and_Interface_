# 📚 Java 객체지향 학습: 상속과 인터페이스

## 1. 추상 클래스와 오버라이딩
* **추상 클래스**: 클래스들의 공통적인 특징을 추출하여 선언한 미완성 클래스이다. `abstract` 키워드를 사용하며, 단독으로 객체를 생성할 수 없고 반드시 상속을 통해 자식 클래스에서 완성해야 한다.
* **오버라이딩(Overriding)**: 부모 클래스나 추상 클래스에 정의된 메서드를 자식 클래스에서 용도에 맞게 재정의하는 것이다. 이를 통해 하나의 인터페이스로 다양한 실행 결과를 내는 다형성을 구현한다.

## 2. 인터페이스(Interface)
* 객체의 사용 방법을 정의한 규격이다.
* 모든 메서드는 기본적으로 추상 메서드이며, 이를 구현하는 클래스는 인터페이스의 모든 메서드를 반드시 재정의해야 한다.
* 다중 구현이 가능하여 클래스 간의 결합도를 낮추고 유지보수성을 높이는 데 핵심적인 역할을 한다.

## 3. 상속(추상 클래스) vs 인터페이스
| 구분 | 상속 (추상 클래스) | 인터페이스 |
| :--- | :--- | :--- |
| **관계** | `is-a` (자식은 부모의 일종이다) | `can-do` (해당 기능을 할 수 있다) |
| **목적** | 변수와 메서드의 재사용 및 확장 | 동일한 동작을 수행하는 규격 준수 |
| **상속/구현** | 단일 상속만 가능 (`extends`) | 다중 구현 가능 (`implements`) |
| **구성 요소** | 일반 필드와 메서드 포함 가능 | 주로 상수와 추상 메서드로 구성 |
